# 動作確認チェックリスト: 認証機能とConfig情報のDBへの永続化

**Date**: 2025-01-27  
**Feature**: 001-auth-config-persistence

## 前提条件の確認

- [ ] CockroachDBクラスターが起動している
- [ ] `DATABASE_URL`環境変数が設定されている
- [ ] データベーススキーマ（`configurations`テーブル）が作成されている
- [ ] Clerkの環境変数が設定されている:
  - [ ] `PUBLIC_CLERK_PUBLISHABLE_KEY`
  - [ ] `CLERK_SECRET_KEY`

## 認証機能の確認

### サインアップ/サインイン
- [ ] サインアップページ（`/sign-up`）にアクセスできる
- [ ] 新しいアカウントを作成できる
- [ ] サインインページ（`/sign-in`）にアクセスできる
- [ ] 既存アカウントでサインインできる
- [ ] ヘッダーにユーザーボタンが表示される

### 認証状態の確認
- [ ] サインイン後、認証状態が維持される
- [ ] ページをリロードしても認証状態が維持される
- [ ] サインアウトが正常に動作する

## 設定保存機能の確認（User Story 1）

### 認証済みユーザーでの動作
- [ ] 認証済みユーザーで設定を作成できる
- [ ] 設定がデータベースに保存される
- [ ] 設定一覧を取得できる（自分の設定のみ表示される）
- [ ] 設定を更新できる
- [ ] 設定を削除できる
- [ ] ブラウザを閉じて再度開いても、設定がアクセス可能である

### データベースの確認
- [ ] CockroachDBでデータが保存されていることを確認
  ```sql
  SELECT * FROM configurations;
  ```
- [ ] `user_id`が正しく保存されていることを確認
- [ ] `config`（JSONB）が正しく保存されていることを確認

## 未認証ユーザーの動作確認（User Story 2）

- [ ] 未認証状態で設定一覧を取得すると、空のリストが返る（401エラーではない）
- [ ] 未認証状態で設定を作成しようとすると、401エラーが返る
- [ ] 未認証状態で設定を更新しようとすると、401エラーが返る
- [ ] 未認証状態で設定を削除しようとすると、401エラーが返る

## データ分離の確認（User Story 4）

### 複数ユーザーでのテスト
- [ ] ユーザーAで設定を作成
- [ ] ユーザーBで設定を作成
- [ ] ユーザーAで設定一覧を取得 → ユーザーAの設定のみ表示される
- [ ] ユーザーBで設定一覧を取得 → ユーザーBの設定のみ表示される
- [ ] ユーザーAがユーザーBの設定IDにアクセスしようとすると、404エラーまたは空の結果が返る
